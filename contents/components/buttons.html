Buttons

Buttons let users understand what action will occur when they click or tap them.

Usage

Buttons make actions visible. They enable users to change or complete a task. An example of their usage is the installation of a new extension. If the action is to navigate to a different page use a link instead of a button with the exception of a back or forward button.

Types

Default

Default buttons when you have to list multiple actions in one single view or for secondary actions when combined with a primary action.

Primary

Primary buttons are used to draw attention to a specific action. Try to limit one primary button per view unless is a reapating primary action.

Ghost

Ghost buttons are used with icons only. Add a background if there is not enough cue that the button is actionable.

Variants

Label

Use a consistent voice with a short and imperative label. Make sure to check the voice and tone guidelines in order to lead users most effectively.

Icon

If there is enough cue that the button is clickable the icon can stand by itself. Otherwise combine it with a background when the context is not clear enough.

Label and icon

When necessary pair a button label with an icon. It enhances the meaning of a call to action and improves recognition.

Dropdown

Dropdowns allow users to select between multiple options when open. When close dropdowns shows the current selction.

Segmented

Segmented buttons are dropdowns with two separate sections. One section displays the current selection. If pressed the action is triggered. The other section, an arrow, if pressed displays the other options.

Footer button

Footer buttons are sticky to the bottom of a component like a doorhanger or a sidebar. The content scrolls and overflows underneath.
